<risk_notes>
## Risk / migration notes

- **Structural guard is the enforcement, docs are the explanation:** The allow-list in `Invoke-AdmanRemoteCimQuery` is what actually prevents second-hop queries. `docs/REMOTE-OPS.md` tells operators why and what to do if they need second-hop later.
- **No CredSSP, ever in v1:** Even if an operator asks for it, CredSSP must not be added to the ladder or query helpers. Any future second-hop work must go through RBCD/JEA design in a later phase.
- **RBCD/JEA are documented only:** Do not implement RBCD configuration, JEA endpoints, or delegation changes in this phase. The doc points operators at the right Microsoft guidance.
- **Firewall guidance is informational:** The tool does not open ports or modify firewalls. The doc gives admins the port facts so they can choose which transport to enable.
- **Sensitive accounts note:** Because Phase 3 performs no delegation, the "Account is sensitive and cannot be delegated" flag has no effect. This is correct and should be stated clearly so operators do not think the tool bypasses it.
</risk_notes>
