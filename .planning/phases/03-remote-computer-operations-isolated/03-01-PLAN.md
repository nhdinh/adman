<objective>
Build the per-host transport probe and process-only cache that Phase 3 remote enrichment depends on. This plan ships `Connect-AdmanTarget` with the fixed WinRM -> CIM/WSMan -> CIM/DCOM -> skip ladder (D-05), a job-wrapped `Test-WSMan` timeout for Windows PowerShell 5.1 (D-02), a process-only transport-name cache keyed by uppercase computer name (D-04), and the two new config timeout keys `transport.timeouts.perHostProbeCap` and `transport.timeouts.totalInventoryRemoteCap` (D-02). Unreachable hosts are reported as `'Skipped'` (D-06, RMT-02). No Public verbs change in this plan.

Purpose: Quarantine transport detection in one Private connector so the inventory report and future live-action verbs never branch on protocol directly.
Output: Working ladder + cache + timeout config + unit-test suite covering ladder order, cache behavior, skipped outcomes, and cap enforcement.
</objective>

<risk_notes>
## Risk / migration notes

- **Config upgrade:** Existing `.store/config.json` files may lack the new timeout keys. Initialize-AdmanConfig merges missing keys from `config/adman.defaults.json` on every load without overwriting existing values, so the tool does not fail-closed on upgrade and user-edited timeout values are preserved. Operators should still verify the defaults (10s per host, 120s total) are appropriate for their fleet before relying on them in production.
- **Live session reuse rejected (D-04):** The cache stores only a transport name string. Do NOT extend it to hold `CimSession` or `PSSession` objects in this phase; session lifetime across firewall sleep/trust changes is explicitly deferred.
- **PowerShell 5.1 timeout reality:** `Test-WSMan` has no native timeout parameter on 5.1. The `Start-Job` wrapper is the chosen cross-edition compromise. It adds per-call overhead but guarantees the menu cannot hang on dead hosts.
- **Parallel deferred:** Serial probes are intentional (D-02). Parallel fleet probes are explicitly out of scope for Phase 3 and must not be introduced here.
- **No live actions:** This plan ships only transport detection. Remote live actions (service restart, gpupdate, etc.) remain v2 scope `RMT-V01` and must not be added.
</risk_notes>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator -> connector | Operator calls inventory report; connector probes targets on their behalf |
| connector -> target host | WinRM/WSMan/DCOM network probes; no credential forwarding beyond OS Kerberos/NTLM |
| connector -> cache | Module-scoped hashtable stores only transport name, no live sessions |
| connector -> caller | Return value is a transport name or 'Skipped'; raw exceptions are caught and translated |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Denial of Service | Test-WSMan on dead host | high | mitigate | Job-wrapped timeout with hard cap; dead hosts return 'Skipped' without hanging the menu (D-02, Pitfall 1) |
| T-03-02 | Information Disclosure | Verbose error messages from target | medium | mitigate | Convert-AdmanRemoteError maps raw HRESULTs/stack traces to short operator strings; no stack trace crosses the connector boundary |
| T-03-03 | Tampering | Cache poisoning via malformed key | low | mitigate | Key is uppercase computer name only; no user input beyond the computer name is accepted; cache value is validated against a fixed set |
| T-03-04 | Elevation of Privilege | Credential reuse on hop host | high | mitigate | No live session objects are cached; no CredSSP transport option exists in Phase 3 (D-07) |
| T-03-SC | Tampering | Built-in module reliance | low | accept | CimCmdlets and Microsoft.WSMan.Management are OS components; no gallery install needed; Package Legitimacy Audit in 03-RESEARCH.md confirms OK verdicts |
</threat_model>
