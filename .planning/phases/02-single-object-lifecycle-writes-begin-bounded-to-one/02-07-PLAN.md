<objective>
Restore the core safety property (SAFE-01/SAFE-02) on the cmdlet path by removing the unconditional `-Confirm:$false` forwarding at every Public mutation verb call site. The forwarding sets `$ConfirmPreference='None'` inside the gate via dynamic scope, which collapses the prompt condition at `Confirm-AdmanAction.ps1:81` and silently disarms confirmation for all 20 mutation call sites — including the typed-count branch for `Remove-LocalUser` (D-03).

Purpose: This is the Phase-2 core safety property. UAT Test 5 showed Disable-AdmanUser and Enable-AdmanUser both executed immediately with no prompt. Test 11 showed the typed-count prompt for Remove-AdmanLocalUser also absent. Both trace to the same root cause: `-Confirm:$false` forwarded into `Invoke-AdmanMutation` at every Public call site.

Output: 17 Public verb files edited (20 call sites total; Set-AdmanUserPassword and Set-AdmanLocalUser each have 3 sites), one new Pester regression test file proving the prompt is restored.
</objective>

    Why this works: when the Public verb is invoked as a plain cmdlet (no `-Confirm:$false` from the caller), the Public verb's own `[CmdletBinding(SupportsShouldProcess, ConfirmImpact='High')]` sets `$ConfirmPreference='High'` in its scope. When the CALLER explicitly passes `-Confirm:$false` to the Public verb, the Public verb's own `$ConfirmPreference` becomes `'None'` and that value flows through dynamic scope to the gate — the prompt is correctly bypassed. Same for `-Force`.
