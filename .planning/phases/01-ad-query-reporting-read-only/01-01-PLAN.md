<objective>
Implement the read-only adman console menu: a single flat while-loop in Start-Adman, a private menu-item table, and a private per-action parameter prompter that routes every selection to the same Public verb a senior calls directly. This plan delivers MENU-01..04 per D-01.

Purpose: The menu is the junior-safe path; the senior path is the same parameterized Public verb, so there is one code path and two speeds.
Output: A working Start-Adman entry point plus unit tests proving dispatch, back/quit navigation, and renderer hand-off.
</objective>

    - PROPERTIES FIELD (Cycle 4 finding — schema source for empty-result renderer output): each menu entry MUST include a `Properties` field whose value is a [string[]] of the D-03 schema column names that verb emits. This is the single source Plan 01-04 Start-Adman uses to pass -Properties to Format-AdmanReport / Export-AdmanReportCsv / Export-AdmanReportHtml so a zero-row report still renders headers. Pin the exact arrays (they mirror the D-03 schema in 01-02 ConvertTo-AdmanResult plus the Bucket column added by report verbs in 01-03):
      - Find-AdmanUser -> @('ObjectType','Name','SamAccountName','Enabled','DistinguishedName','ObjectSid','ObjectGuid','DisplayName','UserPrincipalName','LockedOut','PasswordExpired','PasswordLastSet','AccountExpirationDate','LastLogonDate','whenCreated','whenChanged')

      - Get-AdmanStaleReport -> @('ObjectType','Name','SamAccountName','Enabled','DistinguishedName','ObjectSid','ObjectGuid','DisplayName','UserPrincipalName','LockedOut','PasswordExpired','PasswordLastSet','AccountExpirationDate','LastLogonDate','whenCreated','whenChanged','Bucket')
      - Get-AdmanAccountStateReport -> @('ObjectType','Name','SamAccountName','Enabled','DistinguishedName','ObjectSid','ObjectGuid','DisplayName','UserPrincipalName','LockedOut','PasswordExpired','PasswordLastSet','AccountExpirationDate','LastLogonDate','whenCreated','whenChanged','Bucket')
      - Get-AdmanInventoryReport -> @('ObjectType','Name','SamAccountName','Enabled','DistinguishedName','ObjectSid','ObjectGuid','OperatingSystem','OperatingSystemVersion','OperatingSystemServicePack','IPv4Address','DNSHostName','LastLogonDate','whenCreated','whenChanged','Bucket')
      - Get-AdmanRecoveryPostureReport -> @('RecycleBinEnabled','ForestFunctionalLevel','TombstoneLifetime','Generated','Freshness')
    - DECISION (Cycle 4 — approach chosen): extend the menu metadata schema with a `Properties` field per verb entry (this approach), NOT a separate Get-AdmanReportProperties helper. Rationale: the menu definition is already the single table Start-Adman reads for dispatch; co-locating the output schema with the verb entry keeps the schema discoverable, avoids a second lookup table that can drift from the menu, and lets Plan 01-04 read $entry.Properties directly with no extra function call. The alternative (a Get-AdmanReportProperties -Verb helper) was rejected because it duplicates the verb->schema mapping in a second location.
    - Read one line via Read-Host 'Select'; accept 'Q'/'q' to exit; validate integers 1..N. Top-level reserved inputs are `1..N` and `Q` ONLY — `B` is NOT reserved at the top level (it is reserved inside action prompts per UI-SPEC §Reserved inputs). On bad input at the top level, re-prompt with 'Invalid selection. Enter a number or Q.' (Addresses review LOW: top-level copy must not promise `B`.) The action-prompt invalid-input copy remains 'Invalid selection. Enter a number, B, or Q.' per UI-SPEC.
    - For a valid choice, call Private/Menu/Read-AdmanActionParams.ps1 with the PromptSpec; this helper reads the required inputs and returns a hashtable of parameters. It must honor 'B' (return $null to resume the loop), 'Q' (exit Start-Adman), and validate numeric sub-choices. Empty required input re-prompts once, then treats a second empty as 'B'.
    - If parameters are returned, invoke the Public verb by name (e.g., & $Verb @params) and return/output the PSCustomObject[] directly. Do NOT pipe to Format-AdmanReport here; renderer dispatch belongs to Plan 01-04.
    - The menu must contain no AD read logic and no formatting logic except the startup banner.
    Add a per-file [Diagnostics.CodeAnalysis.SuppressMessage('PSAvoidUsingWriteHost','')] attribute to Start-Adman because PSScriptAnalyzerSettings.psd1 disables PSAvoidUsingWriteHost globally for the TUI module.
