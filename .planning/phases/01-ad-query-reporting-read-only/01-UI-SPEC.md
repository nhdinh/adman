## Spacing Scale

Console spacing is measured in **blank lines** and **space-character indents**. All values are integer multiples of one line or one column.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 0 blank lines / 1 space char | Inline separators, punctuation gaps |
| sm | 1 blank line or 2-space indent | Compact grouping inside a prompt block |
| md | 1 blank line + 4-space indent | Section break inside a menu action |
| lg | 2 blank lines | Between major screen regions (banner / menu / output) |
| xl | 3 blank lines | After a full report before returning to menu |
| 2xl | 4 blank lines | Major session boundaries (startup banner) |
| 3xl | 5 blank lines | Rare; use sparingly |

**Exceptions:**
- Menu item list: single blank line before the prompt, no blank lines between numbered items (compact scanability).
- Capability banner: `lg` (2 blank lines) after the banner before the menu.
- Error blocks: `sm` (1 blank line) before and after the red error line so it does not visually merge with adjacent output.

## Typography

PowerShell cannot control font family, size, or weight. Typography is expressed through **semantic color roles**, **case conventions**, and **decorative ASCII/Unicode characters**.

| Role | Visual Treatment | Rationale |
|------|------------------|-----------|
| Body / Prompt | Default foreground (`White`/`Gray7`) | Default readable text |
| Label / Menu item | Default foreground, plain sentence case | Numbered list items |
| Heading / Section header | `Cyan` foreground, optionally bracketed with `===` | High contrast, draws attention without graphics |
| Display / Banner | `Cyan` or `White` foreground, framed with `=` characters across width | Startup / capability banner |
| Warning | `Yellow` foreground | Non-fatal issues (missing RSAT, rights hint) |
| Error | `Red` foreground, prefixed with `[Error]` | Fatal or blocking conditions |
| Success / OK | `Green` foreground | Capability flags that are true |
| Muted / Secondary | `DarkGray` foreground | Footnotes, paths, timestamps |

**Line treatment:**
- Body line height: single-spaced (`Write-Host` default).
- Heading line height: single-spaced with `lg` blank line above.
- Banner: framed with a line of `=` characters; contents left-aligned; `2xl` blank line below.

**No emoji, no box-drawing characters, no animations.** Keep to the 16-color console palette so rendering is identical on Windows PowerShell 5.1, PowerShell 7.6, Server Core, WinRM, and SSH sessions.

## Color

Console palette only. Use `$Host.UI.RawBackgroundColor` / `$Host.UI.RawForegroundColor` safe `System.ConsoleColor` values. Avoid bright backgrounds.

| Role | ConsoleColor Value | Usage |
|------|--------------------|-------|
| Dominant (60%) | `Black` / host default background | Screen background |
| Secondary (30%) | `DarkGray` | Borders, separators, muted metadata, paths |
| Accent (10%) | `Cyan` | Menu header, prompt prefixes, active selection, report title |
| Warning accent | `Yellow` | Warnings, cautionary notes, "Quit" option highlight |
| Destructive | `Red` | Errors, blocked actions, `[Error]` prefix |
| Success | `Green` | Capability true flags, OK indicators |

**Accent reserved for:**
- Startup banner title and frame
- Top-level menu header (`adman - AD Manager`)
- Prompt prefix text (`Select`, `Enter`, `Output format`)
- Selected/active item indicator
- Report title in HTML `<h1>` and console header

**Destructive (Red) reserved for:**
- Error messages
- "FAIL-CLOSED" blocking messages
- Nothing else in Phase 1 (this phase is read-only)

## Copywriting Contract

All text is plain English, professional, no emoji, no marketing language. Verbs are action-first.

| Element | Copy |
|---------|------|
| Primary CTA (top-level prompt) | `Select: ` |
| Primary CTA (output format prompt) | `Output: ` |
| Empty state heading | `No matching AD objects` |
| Empty state body | `No objects matched the search criteria. Try a different term or verify the managed OU scope.` |
| Error state | `[Error] {problem}. {actionable next step}.` |
| Destructive confirmation | Not applicable â€” Phase 1 is read-only. Destructive actions begin in Phase 2. |
| Back action label | `B. Back` (inside action prompts; returns to top-level menu) |
| Quit action label | `Q. Quit` (top-level) or `Q. Exit` (inside action prompts; exits `Start-Adman`) |
| Invalid selection | `Invalid selection. Enter a number, B, or Q.` |
| Cancelled action | `Cancelled.` |
| Report saved | `Report saved to {Path}` |

### Menu item labels (default ordering)

Top-level flat menu order. Each item is a number plus a short label. `Q. Quit` follows the numbered list.

| # | Label | Dispatches |
|---|-------|------------|
| 1 | Find user | `Find-AdmanUser` |
| 2 | Find computer | `Find-AdmanComputer` |
| 3 | Stale/inactive report | `Get-AdmanStaleReport` |
| 4 | Account-state report | `Get-AdmanAccountStateReport` |
| 5 | Inventory report | `Get-AdmanInventoryReport` |
| 6 | Recovery posture | `Get-AdmanRecoveryPostureReport` |
| Q | Quit | Exits `Start-Adman` |

*Source: default from RESEARCH.md Pattern 1; menu ordering is Claude's discretion per CONTEXT.md.*

### Output format options (numeric, inside action prompts)

When a query/report verb returns results, the menu prompts for renderer choice:

| # | Label | Action |
|---|-------|--------|
| 1 | Console table | `Format-AdmanReport` |
| 2 | CSV file | `Export-AdmanReportCsv` |
| 3 | HTML file | `Export-AdmanReportHtml` |
| 4 | Grid view (if available) | `Out-GridView` / `Out-ConsoleGridView` with `Format-AdmanReport` fallback |
| B | Back | Return to top-level menu |
| Q | Quit | Exit `Start-Adman` |

*Note: Grid view is capability-probed; if unavailable, option 4 is not shown.*

## Interaction Contract

### Reserved inputs

| Context | Reserved Input | Behavior |
|---------|----------------|----------|
| Top-level menu | `Q` or `q` | Exit `Start-Adman` |
| Any action prompt | `B` or `b` | Return to top-level menu (abandon current action) |
| Any action prompt | `Q` or `q` | Exit `Start-Adman` |
| All prompts | Numeric digits | Select by number |
| All prompts | Anything else | Re-prompt with `Invalid selection. Enter a number, B, or Q.` |

*Source: D-01 in CONTEXT.md/RESEARCH.md.*

### Input validation rules

- Top-level selection must be an integer `1..N` where `N` is the current menu count, or `Q`.
- Output-format selection must be `1..4`, `B`, or `Q`.
- `Find-AdmanUser` search-type selection must be `1..3`, `B`, or `Q`.
- Free-text inputs (e.g., `sAMAccountName`) are trimmed but otherwise passed through; the underlying verb validates emptiness.
- Empty input on a required free-text prompt re-prompts once, then treats as `B` (return to menu) on second empty entry.

### Menu-to-verb dispatch (MENU-04)

`Start-Adman` is a thin prompt-and-dispatch layer. It never queries AD or formats output directly. It:
1. Prints the menu.
2. Reads and validates selection.
3. Prompts for the verb's parameters.
4. Calls the Public verb with those parameters (the same call a senior makes).
5. Hands the returned `PSCustomObject[]` to the chosen renderer.

### Renderer behavior

- **Console (`Format-AdmanReport`)**: `Format-Table -AutoSize | Out-String -Width 4096`. Primary renderer everywhere.
- **CSV (`Export-AdmanReportCsv`)**: `Export-Csv -NoTypeInformation -Encoding UTF8`. Prompts for path; validates parent directory exists.
- **HTML (`Export-AdmanReportHtml`)**: `ConvertTo-Html -Head $cssFragment -Title 'adman report'`. Embedded CSS only; no `-CssUri`, no JS, no `-Charset`/`-Meta`/`-Transitional`.
- **Grid picker**: Capability-probed. Desktop edition + interactive + `Out-GridView` available, OR Core edition + `Microsoft.PowerShell.ConsoleGuiTools` installed. On any failure, silently degrades to console table.

*Source: D-04 in CONTEXT.md/RESEARCH.md.*

## Visual Layout

### Startup banner

```text
============================================================
 adman - AD Manager
------------------------------------------------------------
 Domain  : {domain}
 DC      : {DC}
 Recovery: RecycleBin={Yes|No}, FFL={level}
 Freshness: lastLogonTimestamp fresh to within {N} days
------------------------------------------------------------
 {capability flags: green OK, yellow warning, red blocker}
============================================================
```

- Title in `Cyan`, frame in `DarkGray` or `Cyan`.
- Warnings in `Yellow`.
- Blockers in `Red` and terminate startup.
- Followed by `lg` (2 blank lines) then the menu.

### Menu screen

```text
adman - AD Manager

1. Find user
2. Find computer
3. Stale/inactive report
4. Account-state report
5. Inventory report
6. Recovery posture
Q. Quit

Select:
```

- Header `adman - AD Manager` in `Cyan`.
- Single blank line before list, single blank line before prompt.
- No blank lines between numbered items.

### Report header (console)

```text
{Report Title}
Generated: {ISO-8601 timestamp}
Scope    : {ManagedOUs joined}
Freshness: lastLogonTimestamp fresh to within {N} days (sync interval = {X})

{table}
```

- Report title in `Cyan`.
- Metadata in `DarkGray`.
- One blank line between metadata and table.

### Error block

```text

[Error] {message}. {actionable guidance}.

```

- `sm` blank lines before/after.
- Full line in `Red`.

## HTML Embedded CSS Contract

The HTML renderer uses a single here-string CSS fragment passed to `ConvertTo-Html -Head`. No external files, no JavaScript, no PS6+ parameters.

```css
<style>
body {
    font-family: Segoe UI, Arial, Helvetica, sans-serif;
    font-size: 13px;
    color: #222;
    background: #fff;
    margin: 24px;

    font-size: 18px;
    color: #2b579a;
    margin-bottom: 8px;

.metadata {
    color: #555;
    font-size: 12px;
    margin-bottom: 16px;
}

    border-collapse: collapse;
```

**CSS rules:**
- Professional blue/gray palette.
- Zebra striping on even rows.
- Boolean `Enabled`/`LockedOut` cells get `.true` / `.false` classes.
- No hover effects, no animations, no responsive breakpoints.

## Accessibility & Portability

- All interaction is keyboard-only; no mouse required.
- All output must render identically in Windows PowerShell 5.1 and PowerShell 7.6 LTS.
- All output must work over WinRM/SSH remoting and on Server Core.
- `Out-GridView` is optional and degrades to console table on failure.
- `Out-ConsoleGridView` is offered only on PS7 Core when the module is present.
- High-contrast console hosts: color choices map to standard 16-color slots; do not rely on color alone (use `[Error]`, `[No results]`, etc., text markers).
