    - Collect pipeline input into a list. MEMORY BOUND (MEDIUM-4): the console renderer is bounded by the host display buffer; for result sets expected to exceed ~10,000 rows the caller should use Export-AdmanReportCsv instead. Document this in comment-based help.

    - EMPTY-RESULT SCHEMA (MEDIUM, Cycle 2/3 finding): when the pipeline yields zero objects, the renderer cannot infer headers. If $Properties is NOT supplied and the pipeline is empty, emit the literal string '(no results)' and return.     - Do not write files; emit strings to the pipeline/host.
    Create Public/Export-AdmanReportCsv.ps1 for RPT-02:
    - Accept [Parameter(Mandatory,ValueFromPipeline)]$InputObject, [Parameter(Mandatory)]$Path, and an optional [string[]]$Properties used ONLY when the pipeline is empty (see empty-result schema below).
    - STREAMING (MEDIUM-4 + Cycle 2/3 finding): use a begin/process/end structure with explicit first-row handling. Do NOT pipe $InputObject directly to Export-Csv inside process (that overwrites the file on every row and emits one header block per row).
      - begin: set $script:csvFirstRowSeen = $false; validate the parent directory of $Path exists and throw a clear error if not; if a stale file exists at $Path, remove it so reruns do not append onto old data.
      - process: for each $InputObject, if -not $script:csvFirstRowSeen then pipe the object to Export-Csv -Path $Path -NoTypeInformation -Encoding UTF8 (creates the file with headers) and set $script:csvFirstRowSeen = $true; otherwise pipe the object to Export-Csv -Path $Path -NoTypeInformation -Encoding UTF8 -Append (appends a data row, no duplicate header).
      - end: if -not $script:csvFirstRowSeen (empty pipeline), emit a header-only CSV. If $Properties is supplied, write a single header line built from that property list (join with ',', UTF8 with BOM via Out-File -Encoding UTF8) so the file is not zero-byte. If $Properties is NOT supplied, write an empty file (zero bytes) and emit a verbose message that no schema was provided.
    - Memory stays O(1) in the row count because each row is written and released inside process. This is the documented fallback for large result sets (e.g., inventory exports of large OUs).
    Create tests/Render.Tests.ps1 covering console table output, CSV encoding/NoTypeInformation, grid-view fallback on failure, PS 5.1/7 parity, and the following Cycle 2/3 cases:

This proves the first-row/-Append strategy works and rows are not overwritten.
    - CSV empty-result WITH -Properties: pipe zero objects with -Properties @('Name','SamAccountName','Enabled'); assert the file contains exactly one line equal to the joined header and zero data rows.
    - CSV empty-result WITHOUT -Properties: pipe zero objects without -Properties; assert the file exists (zero bytes is acceptable) and no exception is thrown.
    - Console empty-result WITH -Properties: assert the output contains the column names and no data rows.
    - Console empty-result WITHOUT -Properties: assert the output is the literal string '(no results)'.
    - HTML empty-result WITH -Properties: assert the file contains a <table> with a header row matching -Properties and zero <tr> data rows.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Report data -> renderer | D-03 objects only; no raw AD types. |
| Renderer -> filesystem | CSV/HTML files are written to operator-specified paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Information Disclosure | CSV/HTML file written to unexpected path | low | mitigate | Validate parent directory exists; do not create parent directories automatically (fail explicitly so the operator notices the path). |
| T-04-02 | Information Disclosure | HTML report leaks raw AD attributes | low | mitigate | Renderers consume the D-03 schema only; no raw AD object properties are rendered. |
| T-04-03 | Tampering | HTML contains executable JavaScript | low | mitigate | Use ConvertTo-Html -Head with a static CSS fragment; no JavaScript, no external resources (D-04). |
| T-04-04 | Denial of Service | Large result set exhausts memory in pipeline collector | low | mitigate | Export-AdmanReportCsv streams row-by-row (no intermediate collection); Format-AdmanReport and Export-AdmanReportHtml document a ~10,000-row soft bound in comment-based help and direct callers to the CSV renderer for larger sets (MEDIUM-4). |
| T-04-SC | Tampering | Package installs | high | accept | No packages introduced this phase. |
</threat_model>
