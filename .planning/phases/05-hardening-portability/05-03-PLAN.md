Harden the audit subsystem and source-control guardrails: add a per-record SHA-256 hash chain with integrity verifier and rotation, prove event-log escalation on OUTCOME-write failure, and block accidental .store/ commits with a pre-commit hook.

Purpose: The safety model depends on a trustworthy audit log and a clean boundary between runtime secrets/config and source control. This plan implements tamper-evidence, rotation, and defense-in-depth commit protection.
Output: audit.retentionDays config, hash-chain helpers and rotation in Private/Audit/Rotation.ps1, integrated prevHash in Write-AdmanAudit, integrity/event-log/rotation tests, and .githooks/pre-commit.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Audit writer -> filesystem | Hash chain detects tampering after the record is written. |
| Audit writer -> Event Log | Escalation path when the local write fails after a mutation. |
| Developer workstation -> Git | Pre-commit hook blocks accidental secret/config commits. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-03-01 | Tampering | Audit JSONL files | high | mitigate | Implement SHA-256 per-record hash chain with prevHash; provide Get-AdmanAuditIntegrity verifier; document tamper-evident (not tamper-proof) semantics. |
| T-05-03-02 | Repudiation | Write-AdmanAudit OUTCOME path | high | mitigate | Preserve existing fail-closed PENDING throw and event-log escalation on OUTCOME failure; prove with Audit.EventLog.Tests.ps1. |
| T-05-03-03 | Information Disclosure | Audit archive folder | medium | mitigate | Archive folder lives under .store/audit and inherits the same filesystem permissions; access control is OS ACLs, not code. |
| T-05-03-04 | Tampering | .store/ commit guard bypass | medium | mitigate | Pre-commit hook plus CI checkout scan from Plan 05-02; CI scan fails the build if .store/ is present. |
| T-05-03-SC | Tampering | npm/pip/cargo installs | n/a | accept | This plan installs no packages. |
</threat_model>
