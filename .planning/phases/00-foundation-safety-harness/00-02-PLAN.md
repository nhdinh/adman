<objective>
Build the portable, plain-JSON, NON-SECRET configuration substrate: one shared schema, shipped defaults + annotated example, fail-closed load/validate/save/reload, a SID-seeded deny-list, and thin config verbs — all pinned to an explicit path so a stray per-user PSFramework setting can never weaken scope (CONF-01/02/03/05, D-01/D-04/D-05).

Purpose: CONF-02 fail-closed ("refuse writes if managed-OU empty or config/deny-list fails to load") is the second safety boundary after the export boundary; it must be structurally impossible for a missing/empty scope or a PSFramework auto-import to silently permit writes. This plan makes the config layer correct and provable before the gate (00-04) consumes it.

Output: config/ schema+defaults+example, Initialize-AdmanConfig (fail-closed), four config verbs, and four Config.* test files.
</objective>
