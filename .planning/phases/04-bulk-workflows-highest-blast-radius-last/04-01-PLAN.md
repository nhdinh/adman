<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSV file -> bulk engine | Untrusted operator-supplied file crosses into the tool |
| Pipeline search output -> bulk engine | Trusted internal data, still normalized |
| Bulk engine -> Invoke-AdmanMutation | Same-module trusted boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Elevation of privilege | Invoke-AdmanBulkAction against protected accounts | high | mitigate | Every resolved target runs through `Test-AdmanTargetAllowed` before confirmation; refusals are audited as `Refused`. |
| T-04-02 | Tampering | CSV with unknown headers or misspelled action columns | medium | mitigate | `Import-AdmanBulkCsv` allow-lists headers and throws on unknown columns before any gate call (D-25). |
| T-04-03 | Elevation of privilege | Bypassing the cap by padding the input with out-of-scope objects | medium | mitigate | `Assert-AdmanBulkPolicy -EnforceCap` runs after `Test-AdmanTargetAllowed` filtering (D-07). |
| T-04-04 | Denial of service / data loss | Move-ADObject throws on already-in-place object, breaking resume | medium | mitigate | Pre-check parent DN against target path; skip with a Success audit note instead of calling the gate. |
| T-04-05 | Repudiation | Bulk action runs unaudited because audit writer fails | high | mitigate | `Invoke-AdmanMutation` still owns the write-ahead PENDING audit; bulk no-op skips write their own Success record via `Write-AdmanAudit`. |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | Phase 4 installs no packages; no package-manager task exists. |
</threat_model>
