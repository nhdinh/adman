    - `Start-AdmanUserOnboarding -FirstName 'John' -LastName 'Doe'` builds sAMAccountName from `templates.onboarding.NamePattern`, UPN from `sAMAccountName@domain`, and calls `New-AdmanUser`.
    - If any baseline group is protected, the workflow throws BEFORE creating the user or adding any group.
    - If a baseline group add fails, no subsequent group adds run and a Failure audit is written.
    - `-WhatIf` previews the create and all baseline group adds.
  </behavior>
  <action>
    Create Public/Start-AdmanUserOnboarding.ps1:
      - Declare `[CmdletBinding(SupportsShouldProcess, ConfirmImpact='High')]` with mandatory `FirstName` and `LastName` and optional `Force`.
      - Throw the standard WR-01 init message if `$script:Config.ManagedOUs` is missing.
      - Read `$template = $script:Config.templates.onboarding` and validate it has `ParentOuDn`, `BaselineGroups`, and `NamePattern`.
      - Build `$sam = ($template.NamePattern -f $FirstName, $LastName).ToLower()` and `$upn = "$sam@$($script:Config.Domain)"`.
      - For each group in `$template.BaselineGroups`, resolve it via `Resolve-AdmanGroup` and call `Test-AdmanGroupAllowed -Operation 'Add-ADGroupMember'`; throw if any group is refused (D-17).
      - Wrap the workflow in a try/catch: call `New-AdmanUser -Name "$FirstName $LastName" -SamAccountName $sam -UserPrincipalName $upn -ParentOuDn $template.ParentOuDn -Force:$Force -WhatIf:$WhatIfPreference`; then loop baseline groups calling `Add-AdmanGroupMember -Identity $sam -GroupIdentity $g -Force:$Force -WhatIf:$WhatIfPreference`.
      - On catch, write `Write-AdmanAudit -Verb 'Start-AdmanUserOnboarding' -Target $sam -Result 'Failure' -Reason $_.Exception.Message -WhatIf:$WhatIfPreference`, then rethrow (D-13/FLOW-04).
      - Do NOT duplicate password display logic; let `New-AdmanUser` handle D-14 display-once hygiene.
  </action>
