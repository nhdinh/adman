## Overview

adman v1.0 is a menu-driven (interactive TUI) PowerShell toolkit for managing users and computers in an on-prem Active Directory domain. It consolidates AD object lifecycle, reporting & inventory, remote computer operations, and provisioning/offboarding workflows behind one guided interface with strong safety guardrails so a less-experienced admin cannot accidentally damage the directory.

## Milestone Summary

**Key Decisions:**

- Safety-first sequencing: foundation → reads → single writes → remoting → bulk/workflows → hardening
- Config/credential split: portable plain-JSON non-secret config + separate opt-in DPAPI credential file
- "Delete" = disable + quarantine; no hard-delete verb
- All writes route through one non-exported mutation gate (`Invoke-AdmanMutation`)
- Pass-through by default; prompt for domain-admin creds only when rights insufficient
- No hard-coded built-in RID baseline; protection via recursive admin-group membership + custom deny-list
- Dual PowerShell edition support: Windows PowerShell 5.1 + PowerShell 7.6 LTS

**Issues Resolved:**

- Restored confirmation on cmdlet path (removed `-Confirm:$false` forwarding)
- Fixed menu identity/DN resolver for AD identity and OU prompts
- Fixed defensive SID extraction for synthetic pre-create targets
- Closed UAT gap G-05-1 across docs coverage, archive self-hash, and CONF-02 fail-closed message

**Issues Deferred to v2:**

- Saved queries / favorites (RPT-V01)
- Remote live actions (RMT-V01)
- Multiple onboarding templates (FLOW-V01)
- Full idempotent/resume-safe bulk with persisted job state (BULK-V01)
- HR-CSV-driven provisioning (FLOW-V02)
- Read-only GPO reporting (RPT-V02)
- Scheduling of reports/workflows (PLAT-V01)
- Compiled `.exe` distribution (PLAT-V02)
- Elevation/stored-privileged-credential model (PLAT-V03)
- Multi-domain / cross-forest support (PLAT-V04)

**Technical Debt Incurred:**

- PowerShell 7.6 LTS CI verification relies on `mchave3/setup-pwsh` external action (pinned to commit hash)
- Self-signed CI certificate path uses `Cert:\LocalMachine\Root`/`TrustedPublisher` for non-interactive runners
- Config schema test fixture `tests/Config.Schema.Tests.ps1` has a pre-existing gap around required `audit` block
