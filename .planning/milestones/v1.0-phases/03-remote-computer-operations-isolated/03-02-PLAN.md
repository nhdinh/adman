<objective>
Add the read-only remote query layer and wire it into the existing inventory report. This plan ships `Invoke-AdmanRemoteCimQuery` (a guarded, local-only CIM runner), `Invoke-AdmanRemoteQuery` (online/OS/uptime/logged-on user with a hard-capped session-setup probe and shrinking per-operation budget), extends `Get-AdmanInventoryReport` with serial per-row enrichment and total-cap enforcement (D-01, D-02, D-03), updates its comment-based help, and updates the menu label and renderer property list. The query layer is transport-agnostic: it receives a transport name from `Connect-AdmanTarget` and never branches on protocol logic (RMT-03).

Purpose: Deliver the actual remote-enriched fleet view promised by RPT-06 and RMT-03 while keeping transport complexity quarantined in the connector.
Output: Working remote query functions + enriched inventory report + updated menu + unit tests covering query behavior and inventory enrichment.
</objective>

<risk_notes>
## Risk / migration notes

- **Schema contract preserved:** Remote columns are appended by `Get-AdmanInventoryReport` after `ConvertTo-AdmanResult`; the existing D-03 schema contract test in `tests/Result.Schema.Tests.ps1` must NOT be modified. If a renderer breaks because it expected the old column set, the fix is in the renderer or the menu Properties array, not the mapper.
- **Uptime as [TimeSpan]:** Renderers receive a `TimeSpan` object. The existing console renderer will format it as `7.12:34:56`; CSV will emit an ISO-ish duration string; HTML will use the same. No formatter changes are required unless the team wants a rounded-days view.
- **LoggedOnUser limitation:** `Win32_ComputerSystem.UserName` returns only the interactive console session and is often empty on RDP-only servers. This is a documented Windows limitation, not a tool bug; do not add `quser`/`qwinsta` fallbacks in Phase 3 (deferred).
- **Per-host cap now bounds probe + session setup + queries combined:** Get-AdmanInventoryReport starts a per-host stopwatch before Connect-AdmanTarget and passes the remaining budget into Invoke-AdmanRemoteQuery. Invoke-AdmanRemoteQuery hard-caps session setup with Test-AdmanCimSessionTimeout and recomputes the remaining budget before New-CimSession and each Get-CimInstance. A slow host cannot consume more than `perHostProbeCap` seconds across classification, session setup, and the two CIM reads.
- **CIM errors count as Skipped:** When Invoke-AdmanRemoteQuery catches a session-setup or query exception, it returns Transport='Skipped'. The report increments the skipped count and emits the single summary warning. This preserves the first-class non-error outcome (D-06) and makes the skipped count accurate.
- **Total cap semantics:** Once the total cap is reached, all remaining rows are marked 'Skipped' with no further network attempts. Already-probed rows keep their results. This is deterministic and fastest.
- **No audit for reads:** Phase 3 is read-only; SAFE-03 does not require audit records for inventory reads. Do not add read-audit writes here.
</risk_notes>
