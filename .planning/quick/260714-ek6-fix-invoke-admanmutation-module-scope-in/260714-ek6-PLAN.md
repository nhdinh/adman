Fix the CommandNotFoundException for `Invoke-AdmanMutation` in the two lab-only integration test files by invoking the deliberately non-exported gate through module scope — the same pattern the passing unit tests already use.

Purpose: `Invoke-AdmanMutation` is intentionally absent from `FunctionsToExport` (SAFE-08 — the gate is internal-only). Why the wrapper is needed: the gate is not exported (adman.psd1 FunctionsToExport line 53 excludes it per SAFE-08), so after Import-Module it is invisible in test scope. The `& (Get-Module adman) { ... }` scriptblock executes INSIDE the module's session state, where the private function resolves. Because that scriptblock runs in module scope, the test's `$script:` variables (e.g. `$script:TestOu`, `$fixture.DistinguishedName`) are NOT visible inside it — so the target DN(s) must be passed in as a `param($t)` argument. This `& (Get-Module adman) { param($t) ... } $args` pattern works on both PowerShell 5.1 and 7.6.

Edit 1 — tests/Safety.WhatIf.Integration.Tests.ps1, the single call site (currently lines 58-59):
Replace:
    $result = Invoke-AdmanMutation -Verb 'Disable-ADAccount' `
        -Targets @($script:TestOu) -WhatIf -Confirm:$false
With:
    $result = & (Get-Module adman) {
        param($t)
        Invoke-AdmanMutation -Verb 'Disable-ADAccount' -Targets $t -WhatIf -Confirm:$false
    } @($script:TestOu)
(Keep `-WhatIf -Confirm:$false` exactly; pass `@($script:TestOu)` as the trailing argument so it binds to `$t`.)
